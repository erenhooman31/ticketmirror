Current policy

TicketMirror currently stores provider-supplied customer fields from booking emails and applies role-based read/write controls on booking screens. Field-specific requirements can be added here when the data model is ready.

Field group Current behavior
Traveler contact Stored from provider payloads when available; visible according to existing booking permissions.
Participants Stored as provider values and editable operational fields where mutation is permitted.
Provider references Required for stable booking matching and not editable as customer data.
